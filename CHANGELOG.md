## Type system (Track A)

- **A0 Type-algebra inventory** — `Alias`, `Function`, `Object` (structural + index sigs), literal types, `Union`, `Nullable`, `Intersection`, `Negation`, `Unknown`/`Infer`, `Never`, `Type`, `Custom`.
- **A3 Intersection surface syntax** — `A & B` parses, binds tighter than `|`, folds through `intersectType` at parse time; narrowing algebra (`intersectType`) done. Object-type structural intersection: `{a} & {b}` now merges field-wise to `{a, b}` (shared fields refine to their per-field meet; an impossible field → `Never`).
- **A3 fix: `never`-typed value rejected cleanly (no codegen crash)** — forming a VALUE for a binding/annotation whose type folds to `never` (`type C = A & B` with `A` an object and `B` a disjoint string literal) is now a clear, source-located type error — `Type \`C\` is \`never\` (an empty type — \`A & B\`, an object and a string literal, have no common values), so it has no values` — instead of slipping through to codegen and dying as the opaque `Unhandled AST -> WASM "Never" type`. The diagnostic RANGE points at the `A & B` intersection EXPRESSION (a source span the parser captures when it folds the `&` and threads alongside the empty-intersection tag), not the alias name or the value site. `ensureType`'s never branch reports when the TARGET is a (tagged) empty intersection, a real value flows into it, and a real `ctx` is present; legitimate `never` (a narrowed unreachable branch, a membership/narrowing query) is untouched. A defensive backstop in `toWasm.ts`'s `toWasmType` throws a clear compiler diagnostic if a `Never` ever reaches codegen (unreachable in practice). (`tests/cases/types/never-value-*.vl`, `never-narrowing-legit.vl`)
- **A14 fix: honest messages for degenerate self-referential aliases** — a pure self-cycle `type D = D` no longer claims `opaque/recursive type aliases are not yet supported` (which implied a future feature); it reports `Type \`D\` is defined in terms of itself with no base type, so it can never have a value` — it has no structural base, so nothing can inhabit it. A self-referential intersection `type E = A & E` (same class: a definition in terms of itself that collapses to `never`) reports `Type \`E\` refers to itself inside an intersection (\`A & E\`), so it reduces to \`never\``, ranged on the `A & E` expression. LEGITIMATE recursive aliases (`type List = { rest: List[] }`, mutual recursion) are unaffected — they resolve to a real structural body and never hit the degenerate-cycle path. (`tests/cases/types/self-alias-still-clean.vl`, `never-value-self-intersection.vl`, `recursive-array-element.vl`, `mutual-recursive-type.vl`)
- **B17 lint: empty-intersection warning** — a `type X = A & B` (or a binding so annotated) whose two CONCRETE operands share no values folds to `never`; the declaration is flagged with a `warning` (`code: empty-intersection`, `source: vital`) — `Intersection \`A & B\` is empty (\`never\`) — an object and a string literal, have no common values` — ranged on the `A & B` intersection expression (not the alias name), matching the never-value error. A self-referential `type E = A & E` instead warns `Type \`E\` refers to itself inside an intersection (\`A & E\`), so it reduces to \`never\``. Scoped to concrete operands: a generic `T & U` (empty only for some instantiation) is NOT flagged. (`tests/cases/lint/empty-intersection.vl`, `generic-intersection-no-warn.vl`)
- **A4 Negation surface syntax** — `!A` and `x !is T` (Kotlin-style negated guard); `subtractType` narrowing algebra. → `DECISIONS.md`
- **A5 Flow narrowing (broad)** — nullness, union-member, post-guard clauses, `&&`/`||` chains, De Morgan, else-of-else-if, literal discrimination, `?.`/`??`. (→ `docs/narrowing.md`)
- **A6 `is` operator + tagged unions** — `x is T` discriminates a value union at runtime (niche/value-kind/boxed encodings, global tag registry, `coerceUnion`). (→ `docs/unions.md`)
- **A6b Proof-carrying narrowing (degenerate)** — a body that is exactly `return <predicate-on-a-param>` is an inferred guard; guard-clause `if absent(v) { return }` supported.
- **A7 Real `string` type (core)** — proper Object (`{[i32]: i32}` index sig → i32-array of char codes), `.length`/`s[i]`/`+`/`==`/`slice`/`indexOf`/`includes`/`charCodeAt`.
- **A10 Parametric types / generics** — Stage 1 (function type params), Stage 2 (array element inference), Stage 3 (generic `type` aliases incl. nested/array); `map`/`filter` build-side generics over growable `T[]`. (`tests/cases/generics/`)
- **A11 Recursive structural types** — `type Tree = { value, left: Tree | null, … }` constructs/traverses/compiles (cycle-safe traversals + self-referential WasmGC rec-group). Mutual recursion across *separate* `type` decls (`type A = { b: B | null }` ↔ `type B = { a: A | null }`) now resolves: a top-level type-name pre-pass hoists every `type` name before any body is parsed, so a forward/mutual reference stays a lazy `Alias` leaf and both aliases tie into one rec-group. (`tests/cases/types/mutual-recursive-type.vl`) Recursion through an **array element type** (`type List = { value: i32, rest: List[] }`) now compiles too — the element struct, its growable-list `{backing,len,cap}` wrapper, and the backing array are discovered as one strongly-connected component and built in a single WasmGC rec group (forward-referencing each other), instead of unboundedly re-entering type lowering and overflowing the stack. (`tests/cases/types/recursive-array-element.vl`) Recursion through a **map value type** (`type Tree = { value: i32, children: { [string]: Tree } }`) now compiles as well — the map's hash struct and its `vals` backing array join the same rec group, so the value-type back-edge is threaded via forward refs instead of overflowing through `mapType`→`mapArrayType`→`refHeapType`. (`tests/cases/types/recursive-map-value.vl`)
- **A12 Soundness corpus (started)** — must-error/must-not-error corpus under `tests/cases/soundness/`; strict by default; known-unsound corners `xfail`-marked.
- **A13 Operator-constraint inference (core)** — fully-inferred structural functions monomorphize per call shape (i32 & f64 from two call sites).
- **A14 Named/opaque type crash fix** — bodyless-`type Point` infinite-recursion crash fixed (cycle-guarded `getConcreteType`; now errors cleanly). → `DECISIONS.md`
- **A15 Structural equality** — `==`/`!=` by value; functions by reference; arrays + nested structs recurse via `valueEq`/`isEquatable`. → `DECISIONS.md`
- **A16 Literal-union types (type-level)** — annotations constrain (`"a"|"b"` rejects `"c"`); `==`/`!=` discriminate + narrow; covering `if/else if` chain exhaustive. `flattenType` dedup uses an O(n) value-key set when every variant is a literal (the common case; distinct literals never subsume), replacing the O(n²) pairwise `validateType` scan — a left-folded N-member literal union now type-checks in O(n²) not O(n³) (160 members 485ms→3.8ms). Shared front-end logic (carries to `typecheck.vl`). (`tests/cases/types/literal-union-dedup.vl`)
- **A7b `boolean`-where-`i32`-expected coercion** — a `boolean`-typed VALUE coerces to `i32` (true→1, false→0) at an assignment boundary (binding/argument/return); no-op in codegen (shared i32 rep). One-directional (`i32` ⊄ `boolean`); excludes a bare boolean literal and membership/narrowing queries (`x is boolean` on `string | i32` still rejected).
- **A17 Forward / mutual-reference return-type inference** — demand-driven (lazy + memoized, cycle-detected) return-type inference; a function calling a later-defined or mutually-recursive function no longer infers `any`; only a genuinely base-case-less inferred cycle still needs an explicit annotation. (#105; detail: `docs/selfhost-gaps.md` §A17)

(`compiler/builtins/maps.ts`) A map whose VALUE type is the enclosing recursive type (`{ [string]: Tree }`) now compiles — the hash struct + its `vals` array thread into the type's WasmGC rec group (mirrors A11 array-element recursion). (`tests/cases/types/recursive-map-value.vl`)

- **B14 Methods via explicit `self` + UFCS (core)** — free `self`-first function callable as `o.f(args)`; field-then-self-fn resolution; non-`self` functions not instance-reachable. → `DECISIONS.md`

- **B17 Export-aware top-level unused** — exported top-level bindings exempt from unused-variable lint; landed with `export` keyword (H0 phase 1).
- **B17 fix: unused-import diagnostic + remove fix** — an unused imported name now squiggles the imported NAME (`a`, or the `as`-alias `y` in `{ x as y }`), not the `import` keyword (`parseImport` threads each specifier's local-identifier token span to `declareBinding`), and is reported under a dedicated `unused-import` code with the message "Unused import `a` (remove it)". Imports get NO `_`-prefix suggestion (prefixing would require aliasing, not a bare `_`); the only quick-fix removes the specifier — deleting the whole `import` line when it was the sole specifier, else dropping just that specifier and one adjacent comma (first/middle/last handled with no dangling comma; an `{ x as y }` alias clause is removed whole). Import bindings carry an `isImport` flag (`Binding.isImport`, set in `parseImport`) that the lint branches on.
- **B19 `return` / early returns** — early, from loops, fall-through; bare `return` yields null.
- **B-debug Name section + source maps + trap-to-source** — wasm name section + Source Map v3 (binaryen debug locations, survives `optimize()`); trap → precise VL `file:L:C` error message. (#76)
- **B6 Nullable-list narrow-then-use codegen fix** — a `T[] | null` narrowed to non-null (`if xs != null { … }`) then read into a list op (`xs[i]`, `xs[i]=v`, `xs + …`, `for…in`, `map`/`filter`) no longer crashes binaryen's `LocalSubtyping` (`isSubType` assertion): narrowing is type-level only, so the value keeps its nullable `(ref null $list)` slot, and stashing it in the op's non-null `(ref $list)` local was a subtype violation. Each such read now `ref.as_non_null`s the list (the narrowing guarantees non-null). Reproduces non-recursively with `i32[] | null`; found by the A11 array lane. (`tests/cases/types/nullable-list-index.vl`)

- **H-emitProgram ref-element lists / arrays-of-structs-and-unions (G7-ref)** — `emitProgram` now lowers growable lists whose ELEMENT is a reference — `Tok[]` (a struct ref) and `Node[]` (the union box ref), the self-host parser's `P.nodes: Node[]` arena + token stream shapes. For a union-element list `ns[i]` yields the union box, usable with `is`-narrowing (`let n0 = ns[i]; if n0 is A { n0.av }`). One ref-element list per program (struct OR union element), alongside the existing i32 list. 6 new live-instantiation cases (struct push+index-a-field+length, `while`-loop build-then-sum, `T[]` literal, struct index-set, union push+index+`is`-narrow, a `Node[]`-shaped mini arena narrowing each element). No `parser.vl`/`ast.vl` change needed (`S[]`/`N[]` already parse to a `TypeRef` with the `[]` suffix).
- **H-emitProgram logical `&&`/`||`/`!` + value-typed `if` (G4)** — `emitProgram` now lowers the short-circuit logical operators, introducing the first VALUE-TYPED `if` (the existing statement `if` stays VOID, blocktype `0x40`). `&&`/`||` are EXPRESSIONS that yield an i32, so they emit an `if` with an i32 RESULT-TYPE blocktype (`0x04 0x7f`) where both arms leave one i32: `a && b` ≡ `if (a) { b } else { 0 }` (then = `b`, `else 0x05`, else = `i32.const 0`, `end 0x0b`); `a || b` ≡ `if (a) { 1 } else { b }` — mirroring `toWasm.ts`'s `m.if(left, right, i32.const 0)` / `m.if(left, i32.const 1, right)` so the RHS is genuinely SKIPPED when short-circuited. `!a` (logical not) is `i32.eqz` (`0x45`) — a `boolean` rides in an i32, so eqz maps 0→1 / non-zero→0, exactly `a == 0`. 8 new live-instantiation cases (`&&` + `||` truth tables, `!a`, `&&`/`||` short-circuit OBSERVED via a global a skipped helper would have written — confirmed untouched, then written when run, on one shared instance, a char-class predicate `(x > 0) && (x < 10)` driving a branch, and a combined `!(a && b) || c`). The value-typed `if` is reusable for any future value-position `if`. No `parser.vl`/`ast.vl` change needed (`&&`/`||` already parse via the precedence table, `!` via `parseUnary`).
- **H-emitProgram boolean params/locals/returns + BoolLit/CharLit (G3)** — `emitProgram` now accepts `boolean` as a valid scalar wherever i32 is (boolean param in `checkParams`, boolean local in `checkLocalI32`, boolean return via `emitExpr`), treating it as the SAME i32 wasm valtype (`0x7f`) since a `boolean` already rides in an i32 (0/1) — no separate boolean valtype is introduced (`vtKindOfType`/`vtKindOfParam` already fall through to i32). `BoolLit` lowers to `i32.const 1` (true) / `i32.const 0` (false) from the already-decoded `boolVal`; `CharLit` lowers to `i32.const <code point>`, decoding `charText` the SAME way string literals decode (`decodeStr` strips the quotes + resolves escapes, yielding exactly one code point). Comparisons/branches on booleans already worked via i32. This unblocks the ~40+ self-host predicates that return `boolean`. 7 new live-instantiation cases (boolean param returned true→1/false→0, `if true`/`if false` BoolLit conditions, a function returning `boolean` (a comparison) used in a branch, a boolean local driving an `if`, `'a'`→97, and a char local + arithmetic `'A' + 1`→66). f64 stays out of scope. No `parser.vl`/`ast.vl` change needed (`boolean`, `true`/`false`, and char literals already parse).
- **H-emitProgram struct field write + module globals (G2)** — `emitProgram` now lowers struct field WRITES (`s.field = v` → `struct.set $0 <fieldidx>`, `0xfb 0x05`, struct-ref-then-value operand order, reusing the `emitPush` field-store shape) and module-level mutable GLOBALS: top-level `let`/`const` lower to a wasm global section (id 6, `<valtype> 0x01 <constexpr-init> 0x0b`) collected by `collectFns` in declaration order, with bare-identifier reads/writes routed to `global.get`/`global.set` (`0x23`/`0x24`) when the name resolves to a global rather than a local. Globals init via constant expressions only — i32 (`i32.const`), string (`array.new_fixed`), struct (`struct.new`), and `i32[]` (`struct.new` of `array.new_fixed`) literals — so no start function is needed (the `__program__` start-fn fallback for non-constexpr initializers is deferred). - **H-emitProgram string `+`/`==`/slice (G6)** — `emitProgram` now lowers the three string operators the self-host sources lean on (diagnostics build messages with `+`, the lexer keyword tables compare with `==`, lexeme extraction uses `gSrc.slice(start, end)`). A string is the SAME `(array (mut i32))` of code points, so all three lower INLINE (no helper functions, unlike `toWasm.ts`'s `__string_eq__`/`__string_slice__`): `a + b` allocates a new array of `len(a)+len(b)` (`array.new_default`, `0xfb 0x07`) and `array.copy`s (`0xfb 0x11`) a into `[0,lenA)` and b into `[lenA, lenA+lenB)`; `a == b`/`a != b` are ELEMENT-WISE VALUE-equality (a value-typed `block`/`loop`: `false` on a length mismatch, else a per-code-point compare loop, all-equal → 1; `!=` adds `i32.eqz`) — a correctness FIX, since the prior i32 `==` path would have emitted `i32.eq` = WRONG ref-identity for strings; `s.slice(start, end)` clamps each bound JS-style (negative-from-end then `[0,len]`, via `select` `0x1b`) and `array.copy`s the half-open range into a fresh array. Only STRING-typed operands take these paths (via the existing `exprString` tracking) — i32/other-type `+`/`==`/`!=` are unchanged. A shared per-function scratch frame (3 string refs + 4 i32 temps, reserved after the push triples when `fnHasStrOp`) stashes operands so a NESTED string op completes before any scratch write; `exprString` now also recognizes string concat + `.slice` as string-producing (so `let s = a + b` / `let s = x.slice(…)` get the string ref valtype). 14 new live-instantiation cases (concat `.length` + index checks, concat of two locals, `==` same/different-content/different-length, value-equality of two BUILT-UP strings (not ref identity), `!=`, `.slice` compared with `==` + length/index + out-of-range clamp, keyword-table `if word == "let"` dispatch hit + miss, concat built in a `while` loop). `.indexOf`/`.includes`/`.charCodeAt`/`fromCodePoint` and the UTF-8 `array i8` storage migration (B7) stay deferred. No `parser.vl`/`ast.vl` change needed (`+`/`==`/`!=` parse via the precedence table, `.slice(…)` as a member-call).
- **H-emitProgram string-keyed maps via parallel lists (G8)** — `emitProgram` now lowers `{[string]: i32}` maps — the shape the self-host checker (`typecheck.vl`) uses for its scope chain (`Map()`, `m[k] = v`, `m.has(k)`, `m[k] ?? -1`). DELIBERATELY the SIMPLEST correct rep, NOT `toWasm.ts`'s open-addressing hash map: a map is two PARALLEL growable lists + LINEAR SCAN — a 2-field struct `{ keys: (ref $keysList), vals: (ref $i32List) }` reusing the G6/G7 `{backing,len,cap}` wrapper twice (a string-ref keys list over `(array (mut (ref null $strArr)))`, and the existing i32 vals list), with NO hashing/probing/new wasm primitives. `Map()` builds two empty lists boxed into the struct; `m[k]=v` scans `keys` via element-wise string `==` (the just-merged G6 compare) and overwrites `vals[i]` on a hit else `keys.push(k); vals.push(v)`; `m[k] ?? d` is a FUSED scan-then-`vals[i]`-or-`d`; `m.has(k)`/`k in m` scans → 1/0. Absent-key semantics match the checker's usage (it reads every scope map through `?? -1`, guarded by `.has`). A per-function 9-slot map scratch frame (map/key/cur refs, scan + char cursors, found index, two grow-backing refs) is reserved after the string-op frame when a function touches a map; scans are O(n) but the checker's scopes are small so linear is correct and fast. `compiler/parser.vl` gained the `{ [string]: T }` index-signature TYPE annotation (a synthetic `"{[K]:V}"` `TypeRef` name, mirroring how `i32[]` carries its suffix) and the `??` operator (precedence table); `ast.vl` needed no new node (`??` is a plain `BinExpr`, `Map()`/`m[k]`/`m.has` already parse). 6 new live-instantiation cases (set+get, two keys, overwrite-in-place, missing-key `??` default, `.has` present/absent, a scope-chain bind-then-lookup pattern). Iteration/`.keys()`/`.values()`/`delete`/`.pop` are deferred (the checker uses none). With G8 the full annotated self-host front end is now emitProgram-compilable.
- **H-emitProgram maps → real open-addressing hash map (G8)** — `emitProgram`'s map lowering is now a REAL ordered open-addressing hash map (FNV-1a + linear probing + load-factor-1/2 resize), mirroring the host's `compiler/builtins/maps.ts` and SUPERSEDING the parallel-lists/linear-scan placeholder above (which stays in history as a fallback). The map struct grows to 5 fields `{ keys, vals, index, count, size }`: `keys`/`vals` remain the insertion-ordered entry lists (so order is preserved and the G6/G7 grow machinery is reused for append), `index` is a raw i32 array of hash slots → entry-position + 1 (0 = empty, initial cap 8), and `count`/`size` track appended/live entries. `Map()` allocates the lists + an 8-slot zero index; every op HASHES the key (FNV-1a over its code points → non-negative i32, `emitMapHashOf`) into `index` and LINEAR-PROBES (wrapping, `emitMapProbe`) to a free-or-matching slot, comparing keys with the G6 element-wise `==` — no O(n) scan. `m[k]=v` overwrites `vals[entry-1]` on a hit else appends an ordered entry, links `index[slot]=count+1`, bumps count/size, and RESIZES (`emitMapResize`) when `size*2 ≥ cap`: a fresh `2×` index is rebuilt from the ordered entries (hash + inner-probe each, keys/vals untouched). `m[k] ?? d` / `m.has(k)` read the probe result. A per-function 12-slot scratch frame (map/key/cur refs, probe slot/cap/entry, two grow backings, char cursor, hash accumulator, resize index backing + inner slot) replaces the old 9-slot frame. 2 new resize-proving cases (ten keys forcing TWO rehashes with every key looked up; resize interleaved with overwrites + a missing-key probe), atop the 6 original G8 behavior tests — all through real `WebAssembly.instantiate`. DELETE/tombstones (the host's `live` array + compaction) are DEFERRED: the checker's op set has no delete, and the struct shape stays forward-compatible for adding a `live` array later. No `parser.vl`/`ast.vl` change.
